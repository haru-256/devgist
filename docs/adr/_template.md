## 書き方のガイド

* **最初に結論を書く**
  * ADR は「何を選んだか」がすぐ分かるようにする
  * `Conclusion (結論)` セクションで、採用案・初期構成・再検討条件を短く要約する
* **比較表と文章の両方を書く**
  * 表で全体像を示し、本文で判断理由を補足する
* **採用理由だけでなく、却下理由も残す**
  * 後から見返したときに、なぜ別案を選ばなかったか分かるようにする
* **将来の再検討条件を明示する**
  * 「今は採用しないが、どの条件なら見直すか」を残す
* **インフラだけでなく運用・アプリ設計の前提も書く**
  * 例: idempotent、checkpoint、rate limit、重複排除、shard 設計
